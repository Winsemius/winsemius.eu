Processing Pipeline​

Source PDFs are processed into combined markdown "buckets" (1-20 files) optimized for:

  • GPT Assistant upload (< 20MB per file)
  • Semantic categorization by topic
  • Structure preservation (TOC, tables, pages)

See PDF_WORKFLOW.md for details on the processing pipeline.

Usage in Policy Documents​

When citing these sources in policy documents:

  1. Attribution — Include document title and author
  2. Date — Note publication date
  3. Page reference — Cite specific pages when applicable
  4. Context — Explain relevance to policy position

Future Development​

As the policy framework evolves, source materials will be:

  1. Processed into buckets using tools/process_pdfs.sh
  2. Uploaded to GPT Assistant for policy Q&A
  3. Linked from policy sections for deeper context
  4. Updated regularly as new policy documents emerge
